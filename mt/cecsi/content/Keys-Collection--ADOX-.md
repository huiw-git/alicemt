  <introduction>
    <para>Contains all <legacyLink xlink:href="55f116fe-4d56-4892-bffe-0cdd6fc727c9">Key</legacyLink> objects of a <legacyLink xlink:href="a6d74000-0828-49ba-850a-63da865f8802">table</legacyLink>.</para>
  </introduction>
  <languageReferenceRemarks>
    <content>
      <para>The <legacyLink xlink:href="215a5391-f422-42ec-99ea-4e6fbb5d3d64">Append</legacyLink> method for a <legacyLink xlink:href="cdb31c76-e559-475c-b33a-aac24f73e70e">Keys collection</legacyLink> is unique for ADOX. You can:  </para>
      <list class="bullet">
        <listItem>
          <para>Add a new key to the collection with the <legacyLink xlink:href="215a5391-f422-42ec-99ea-4e6fbb5d3d64">Append</legacyLink> method.</para>
        </listItem>
      </list>
      <para>The remaining properties and methods are standard to ADO collections. You can:  </para>
      <list class="bullet">
        <listItem>
          <para>Access a key in the collection with the <legacyLink xlink:href="e11484bb-c5c7-42d8-9bb8-21572125d727">Item</legacyLink> property.</para>
        </listItem>
        <listItem>
          <para>Return the number of keys contained in the collection with the <legacyLink xlink:href="da9ccd1f-d402-41a2-940c-45556fc5340d">Count</legacyLink> property.</para>
        </listItem>
        <listItem>
          <para>Remove a key from the collection with the <legacyLink xlink:href="e6b6e3a4-8952-4d79-81f4-51019c338374">Delete</legacyLink> method.</para>
        </listItem>
        <listItem>
          <para>Update the objects in the collection to reflect the current database's schema with the <legacyLink xlink:href="089b7ca7-684f-4259-8032-5bd1ecc54426">Refresh</legacyLink> method.</para>
        </listItem>
      </list>
